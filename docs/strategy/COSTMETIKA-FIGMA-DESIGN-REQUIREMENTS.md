### 7.2 Naming Conventions
**File Naming Structure**:
`[Category]-[Type]-[Variation]-[Size].png`

**Examples**:
- `IG-Carousel-Education-Slide1-1080x1080.png`
- `Story-LivePromo-Countdown-1080x1920.png`
- `Notion-Cover-Dashboard-1500x600.png`
- `Blog-Featured-SPF101-1200x630.jpg`
- `Email-Header-WeeklyNewsletter-600x200.jpg`

**Layer Naming in Figma**:
- Use descriptive names: "Headline Text", "Product Image Placeholder"
- Group related layers: "Background Group", "CTA Section"
- Avoid generic names: "Rectangle 1", "Text Layer"

## 8. Production Workflow

### 8.1 Design ‚Üí Notion Handoff Process
1. **Designer creates asset in Figma**
2. **Export at specified dimensions and format**
3. **Upload to shared Google Drive folder** (organized by type)
4. **Copy file URL**
5. **Paste URL into Notion database** (in "Link to Figma" or "Asset" property)
6. **Update "Asset Status" to "Ready"**

## 10. Quality Assurance Checklist

**Before Exporting Any Asset**:
- [ ] Brand colors used correctly (hex codes match)
- [ ] Fonts match brand typography guidelines
- [ ] Logo placement follows clear space rules
- [ ] Text is readable at target size (especially for mobile)
- [ ] High contrast for accessibility (WCAG AA minimum)
- [ ] Proper dimensions for platform
- [ ] File size optimized (web assets < 200KB when possible)
- [ ] Transparent backgrounds where needed
- [ ] All text layers converted to outlines (for final exports)
- [ ] File named correctly following naming conventions
